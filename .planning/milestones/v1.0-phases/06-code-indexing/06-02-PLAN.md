interface SyntaxNode {
  type: string;                    // node type name (e.g., "function_declaration")
  text: string;                    // full source text of this node
  namedChildren: SyntaxNode[];     // child nodes (named only, excludes punctuation)

  startIndex: number;              // byte offset into source

  childForFieldName(name: string): SyntaxNode | null;
  parent: SyntaxNode | null;
}

  **Types and helpers to define in extractor.ts:**

  ```typescript
  export interface SymbolExtraction {
    symbol_name: string;
    symbol_type: string;     // "function", "class", "method", "interface", "type_alias", "enum", "constant", "struct", "trait", "impl", "module"
    scope_chain: string;     // dot-notation: "MyClass.save"
    content: string;         // raw source text of the symbol
    start_line: number;      // 1-based
    end_line: number;        // 1-based
    is_overview: boolean;    // true for class/struct/impl overview chunks (no method bodies)
  }

  export interface ExtractionResult {
    symbols: SymbolExtraction[];
    imports: string[];       // raw import path strings for relationship extraction
    exports: string[];       // exported symbol names
  }
  ```

  **Helper functions:**

  1. `buildContextHeader(filePath: string, symbolType: string, scopeChain: string): string`
     - Returns `"File: ${filePath} | ${symbolType}: ${scopeChain}"`
     - Per CODE-04 requirement

  2. `splitLargeChunk(symbol: SymbolExtraction, maxLines: number = 200): SymbolExtraction[]`
     - If `end_line - start_line + 1 <= maxLines`, return `[symbol]` unchanged
     - Otherwise split content into parts of ~maxLines with ~20 line overlap
     - Label each part: `symbol_name` becomes `"${symbol_name} (part ${i}/${total})"`
     - Adjust start_line/end_line for each part
     - Preserve scope_chain, symbol_type, is_overview=false for all parts
     - Use character-based splitting: estimate chars per line from content, split at line boundaries

  3. `getNodeName(node: SyntaxNode): string`
     - Try `node.childForFieldName("name")?.text`
     - Fallback: for arrow functions in `lexical_declaration`, get the variable name from the parent
     - Fallback: return `"<anonymous>"`

  **TypeScript/TSX extraction — `extractTypeScript(root: SyntaxNode, source: string): ExtractionResult`:**

  Walk `root.namedChildren` recursively. Handle these node types:

  1. **`function_declaration`** → symbol_type="function", name from `childForFieldName("name")`
  2. **`class_declaration`** → TWO outputs:
     - Overview chunk: symbol_type="class", content = class signature (first line through opening brace or first ~5 lines + docstring/decorators). Set `is_overview=true`.
     - Method chunks: walk `class_body` children for `method_definition` nodes. Each becomes symbol_type="method" with scope_chain=`"${className}.${methodName}"`.
  3. **`method_definition`** (only when inside class_body) → symbol_type="method"
  4. **`lexical_declaration`** (const/let with arrow_function) → Check if the initializer is an `arrow_function`. If yes: symbol_type="function", name from declarator name. If not an arrow function: symbol_type="constant", extract the const declaration.
  5. **`interface_declaration`** → symbol_type="interface"
  6. **`type_alias_declaration`** → symbol_type="type_alias"
  7. **`enum_declaration`** → symbol_type="enum"
  8. **`export_statement`** → unwrap the declaration inside and process it (may contain function_declaration, class_declaration, etc). Also check for re-exports: `export { x } from './y'` — add './y' to imports list.
  9. **`import_statement`** → extract the module path string (strip quotes), add to imports. Do NOT create a symbol chunk for imports.

  For exports: track symbol names that appear inside `export_statement` or have `export` keyword.

  **Scope chain:** Pass `parentScope: string` through recursive calls. Top-level symbols get just their name. Class methods get `"ClassName.methodName"`.

  **Test cases for TypeScript extraction:**
  ```typescript
  // Test 1: Simple function
  const src1 = `export function greet(name: string): string { return "Hello " + name; }`;
  // Expect: 1 symbol, type="function", name="greet", scope_chain="greet"

  // Test 2: Class with methods
  const src2 = `

      // save logic
    }
    find(id: string): User | null {
      return null;
    }
  }`;
  // Expect: 3 symbols — 1 class overview + 2 methods
  // Class: type="class", name="UserRepo", is_overview=true
  // Methods: type="method", scope_chain="UserRepo.save", "UserRepo.find"

  // Test 3: Arrow function const
  const src3 = `const handler = async (req: Request): Promise<Response> => { return new Response("ok"); };`;
  // Expect: 1 symbol, type="function", name="handler"

  // Test 4: Interface + type alias + enum
  const src4 = `
  interface Config { db: string; }
  type Status = "active" | "inactive";
  enum Color { Red, Green, Blue }
  `;
  // Expect: 3 symbols with correct types

  // Test 5: Import extraction
  const src5 = `
  import { foo } from "./utils";
  import bar from "../lib/bar";
  export { baz } from "./baz";
  `;
  // Expect: imports = ["./utils", "../lib/bar", "./baz"]

  // Test 6: Private constant
  const src6 = `const MAX_RETRIES = 3;`;
  // Expect: 1 symbol, type="constant", name="MAX_RETRIES"

  // Test 7: Context header
  // buildContextHeader("src/utils.ts", "function", "greet") === "File: src/utils.ts | function: greet"
  ```

  **What to AVOID:**
  - Do NOT create chunks for import_statement nodes — they are metadata only
  - Do NOT skip `export_statement` — unwrap and process the declaration inside it
  - Do NOT forget to handle `export default function/class`
  - Do NOT hardcode line numbers in tests — compute from source

  **Python extraction — `extractPython(root: SyntaxNode, source: string): ExtractionResult`:**

  Walk `root.namedChildren`. Handle:
  1. **`function_definition`** → symbol_type="function", name from `childForFieldName("name")`
  2. **`class_definition`** → TWO outputs:
     - Overview chunk: class signature + docstring (first string_expression child or first `expression_statement` containing a string). Set `is_overview=true`.
     - Method chunks: walk class body for `function_definition` children. Each becomes symbol_type="method" with scope_chain=`"ClassName.method_name"`.
  3. **`decorated_definition`** → Unwrap: the `function_definition` or `class_definition` inside. Include decorator text in the content.
  4. **`import_statement`** → plain `import foo` — add "foo" to imports (but these are external, will be filtered later by import-resolver)
  5. **`import_from_statement`** → `from X import Y` — extract module name. Handle:
     - Absolute: `from mypackage.utils import helper` → add "mypackage.utils" to imports
     - Relative: `from . import utils` or `from ..models import User` → add the raw string (e.g., ".utils", "..models") to imports
     - Wildcard: `from module import *` → add module name to imports

  For top-level assignments (variable declarations at module scope), treat as constants if they are simple assignments (`expression_statement` containing `assignment` where right side is a literal or call).

  **Rust extraction — `extractRust(root: SyntaxNode, source: string): ExtractionResult`:**

  Walk `root.namedChildren`. Handle:
  1. **`function_item`** (top-level) → symbol_type="function", name from `childForFieldName("name")`
  2. **`struct_item`** → symbol_type="struct", overview chunk with `is_overview=true`
  3. **`enum_item`** → symbol_type="enum"
  4. **`trait_item`** → TWO outputs:
     - Overview chunk: trait signature
     - Method chunks: walk trait body for `function_item` children (trait method declarations). scope_chain=`"TraitName.method"`.
  5. **`impl_item`** → TWO outputs:
     - Overview chunk: `impl X for Y` or `impl X` signature. symbol_type="impl", name = the type being implemented (or "Type for Trait" if trait impl).
     - Method chunks: walk impl body for `function_item` children. symbol_type="method", scope_chain=`"ImplType.method_name"`.
  6. **`type_item`** → symbol_type="type_alias"
  7. **`const_item`** → symbol_type="constant"
  8. **`static_item`** → symbol_type="constant"
  9. **`use_declaration`** → add the use path text (e.g., "crate::models::User") to imports
  10. **`mod_item`** → if it has NO body (just `mod submodule;`), add submodule name to imports (it's a file reference). If it HAS a body (`mod inner { ... }`), recurse into it as a scope container.

  **Dispatch function — `extractSymbols(root: SyntaxNode, source: string, language: string, filePath: string): ExtractionResult`:**
  - Switch on language: "typescript" → extractTypeScript, "python" → extractPython, "rust" → extractRust
  - After extraction, apply `splitLargeChunk` to every symbol that exceeds 200 lines
  - Prepend context header to each symbol's content: `buildContextHeader(filePath, sym.symbol_type, sym.scope_chain) + "\n\n" + sym.content`
