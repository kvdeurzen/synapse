<feature>
  <name>Conventional Commit Validation Hook</name>
  <files>packages/framework/hooks/conventional-commit.js, packages/framework/test/unit/conventional-commit.test.ts</files>
  <behavior>
    Valid commits that PASS:
    - "feat(auth): add JWT signing" -> pass
    - "fix(session): handle expired tokens" -> pass
    - "refactor(db): normalize query patterns" -> pass
    - "docs(readme): update install instructions" -> pass
    - "test(auth): add login edge cases" -> pass
    - "chore(deps): bump dependencies" -> pass
    - "style(lint): fix formatting" -> pass
    - "perf(query): optimize N+1 queries" -> pass
    - "ci(pipeline): add staging deploy" -> pass
    - "build(docker): multi-stage build" -> pass

    Invalid commits that are DENIED:
    - "add login" (no type or scope) -> deny with format guidance
    - "feat: add login" (missing scope) -> deny
    - "feat(auth) add login" (missing colon after scope) -> deny
    - "feat(AUTH): add login" (uppercase scope) -> deny
    - "feat(auth): add login [task:abc123]" (has [task:id] suffix) -> deny
    - "yolo(auth): whatever" (invalid type) -> deny
    - Long first line exceeding 72 chars -> deny

    Edge cases that PASS silently (fail-open):
    - tool_name is "Read" (not Bash) -> pass silently
    - Bash command is "ls -la" (not git commit) -> pass silently
    - Bash command is "git status" (git but not commit) -> pass silently
    - Malformed JSON input -> pass silently (fail-open)
    - Empty input -> pass silently

    Special cases:
    - HEREDOC-style commits: `git commit -m "$(cat <<'EOF'\nfeat(auth): add login\nEOF\n)"` -> extract first line, validate
    - Multi-line -m with body: `git commit -m "feat(auth): add login\n\nDetailed body here"` -> validate first line only
  </behavior>
  <implementation>
    Hook reads stdin JSON, checks tool_name === "Bash", checks command contains "git commit",
    extracts first line of commit message, validates against regex:
    /^(feat|fix|refactor|test|docs|chore|style|perf|ci|build)\([a-z0-9-]+\): .{1,}/
    Also checks first line <= 72 chars and no [task:...] suffix.
    All errors fail-open (catch -> exit(0)).
    Deny writes JSON with permissionDecision: "deny" and helpful reason.
  </implementation>
</feature>
