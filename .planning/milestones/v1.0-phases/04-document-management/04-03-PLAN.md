<objective>
Build query_documents, update_document, and delete_document tools. These three tools complete the core CRUD surface for documents — query for browsing/filtering, update for metadata changes without re-embedding, and delete for both soft (archive) and hard removal.

Purpose: query_documents enables agents to browse and filter documents by category, phase, status, tags, and priority without triggering any embedding calls (DOC-05). update_document lets agents change document metadata and lifecycle state without expensive re-chunking/re-embedding (DOC-06). delete_document supports both archiving (soft-delete) and complete removal (hard-delete) with cascade to related chunks and relationships (DOC-07). All three enforce lifecycle states and log mutations.

Output: Three new tool files with tests, updated server.ts with all three registered.
</objective>

**Core function `queryDocuments()`:**

Build SQL predicate from filters (AND combination per CONTEXT.md):

```typescript
export async function queryDocuments(
  dbPath: string,
  projectId: string,
  args: QueryDocumentsArgs,
): Promise<QueryDocumentsResult>
```

Flow:
1. Connect to LanceDB, open `documents` table
2. Build WHERE predicate parts:
   - Always: `project_id = '${projectId}'`
   - If category: `AND category = '${category}'`
   - If status: `AND status = '${status}'`
   - If phase: `AND phase = '${phase}'`
   - If tags: `AND tags LIKE '%|${tag}|%'` (pipe-delimited per Research Pitfall 4)
   - If priority: `AND priority >= ${priority}`
   - Always exclude superseded: `AND status != 'superseded'` (superseded docs are old versions — don't return them by default unless explicitly filtered)
3. Execute query: `table.query().where(predicate).limit(args.limit).toArray()`
4. Map results: for each row, create a document summary:
   - `summary`: truncate `row.content` to first 400 characters, append `'...'` if truncated (per Research Open Question 2: ~400 chars ≈ ~100 tokens)
   - Select only the fields listed in the result type (not full content)
5. Return `{ documents, total: documents.length }`

**Summary truncation helper:**
```typescript
function makeSummary(content: string, maxChars = 400): string {
  if (content.length <= maxChars) return content;
  return content.slice(0, maxChars) + "...";
}
```

**Important design notes:**
- NO embedding calls — query_documents is pure metadata scan (DOC-05)
- Tags filter: validate tag input to prevent SQL injection. Tags should match `^[a-zA-Z0-9_-]+$`. Reject tags with special characters.
- Superseded documents are hidden by default. If the caller explicitly filters `status: "superseded"`, they get superseded docs back. The filter logic: if `args.status` is NOT provided, add `status != 'superseded'`; if `args.status` IS provided, use their value exactly.

```typescript
const UpdateDocumentInputSchema = z.object({
  project_id: z.string().regex(/^[a-z0-9][a-z0-9_-]*$/),
  doc_id: z.string().min(1),
  status: z.enum(VALID_STATUSES).optional(),

  tags: z.string().optional().describe("Pipe-separated tags, e.g. '|typescript|backend|'"),
  priority: z.number().int().min(1).max(5).optional(),
  force: z.boolean().optional().default(false).describe("Force archive on carry-forward categories"),
});
```

**Core function `updateDocument()`:**

```typescript
export async function updateDocument(
  dbPath: string,
  projectId: string,
  args: UpdateDocumentArgs,
): Promise<UpdateDocumentResult>
```

Flow:
1. Connect to LanceDB, open `documents` table
2. **Verify document exists (Research Pitfall 3):** Query for doc_id + project_id, limit 1. If not found, return `{ error: 'DOC_NOT_FOUND', message: 'Document {docId} not found in project {projectId}' }`
3. **Lifecycle state transition validation (DOC-09):**
   If `args.status` is provided:
   - Get current status from the existing row
   - Validate transition is allowed:
     - draft -> active, approved (skip forward allowed)
     - active -> approved, archived
     - approved -> archived
     - archived -> active (reactivation)
     - superseded -> (no transitions — superseded is terminal for version management)
   - If transition is invalid, return `{ error: 'INVALID_TRANSITION', message: 'Cannot transition from {current} to {requested}' }`
4. **Carry-forward guard (DOC-10):**
   If `args.status === 'archived'` and current document is a carry-forward category:
   - If `args.force !== true`, return `{ error: 'CARRY_FORWARD_PROTECTED', message: 'Cannot archive carry-forward category {category}. Use force=true to override.' }`
   - If `args.force === true`, allow the transition (user explicitly overriding)
5. **Build update values object:** Only include fields that are provided:
   ```typescript
   const values: Record<string, unknown> = { updated_at: new Date().toISOString() };
   if (args.status) values.status = args.status;
   if (args.phase !== undefined) values.phase = args.phase;
   if (args.tags !== undefined) values.tags = args.tags;
   if (args.priority !== undefined) values.priority = args.priority;
   ```
6. **Execute update:** `await table.update({ where: predicate, values })`
   - Predicate: `doc_id = '${docId}' AND project_id = '${projectId}' AND status != 'superseded'`
   - Only update the LATEST version (not superseded rows)
7. **Verify update took effect (Pitfall 3):** Count rows matching the updated state. If 0, something went wrong.
8. **Log activity:** `logActivity(db, projectId, 'update_document', docId, 'document', { fields_updated: Object.keys(values).filter(k => k !== 'updated_at') })`
9. Return `{ doc_id: docId, updated_fields: Object.keys(values).filter(k => k !== 'updated_at') }`

**CRITICAL: No re-embedding.** update_document NEVER calls embed() or touches doc_chunks. It only modifies the documents table rows.

```typescript
const DeleteDocumentInputSchema = z.object({
  project_id: z.string().regex(/^[a-z0-9][a-z0-9_-]*$/),
  doc_id: z.string().min(1),
  hard: z.boolean().optional().default(false).describe("If true, permanently removes document. If false, sets status to 'archived'."),

});
```

**Core function `deleteDocument()`:**

Flow:
1. Connect to LanceDB, open `documents` table
2. **Verify document exists:** Query for doc_id + project_id. If not found, return DOC_NOT_FOUND error.
3. **Carry-forward guard (DOC-10):** If not `args.force`, check category. If carry-forward, return CARRY_FORWARD_PROTECTED error.
4. **Soft delete (args.hard === false):**
   - `await table.update({ where: predicate, values: { status: 'archived', updated_at: now } })`
   - Log activity: action='archive_document'
   - Return `{ doc_id, action: 'archived' }`
5. **Hard delete (args.hard === true):**
   - Delete from documents table: `await table.delete(\`doc_id = '${docId}' AND project_id = '${projectId}'\`)`
   - Delete from doc_chunks table: `await docChunksTable.delete(\`doc_id = '${docId}' AND project_id = '${projectId}'\`)`
   - Delete from relationships table (both directions): `await relTable.delete(\`(from_id = '${docId}' OR to_id = '${docId}') AND project_id = '${projectId}'\`)`
   - Log activity: action='hard_delete_document', metadata includes chunks and relationships deleted
   - Return `{ doc_id, action: 'hard_deleted' }`
