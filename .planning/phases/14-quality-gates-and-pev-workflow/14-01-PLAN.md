<objective>
Create three PreToolUse enforcement hooks (tier-gate, tool-allowlist, precedent-gate) with fail-closed error handling, unit tests, and settings.template.json wiring.

Purpose: These hooks are the safety net preventing agents from exceeding their authority. Tier-gate enforces decision tier boundaries. Tool-allowlist constrains which Synapse MCP tools each agent can call. Precedent-gate injects context reminding agents to check existing decisions before creating new ones. All enforcement hooks fail-closed (deny on any error).

Output: Three new hook scripts in hooks/, a comprehensive test file proving denial/allow/error behaviors, and updated settings.template.json with PreToolUse matchers.
</objective>

    - tool-allowlist.js: allows when tool is in actor's allowed_tools

    - precedent-gate.js: exits 0 silently on non-store_decision tools
  </behavior>
  <action>
    Create three ESM hook scripts following the established pattern from synapse-audit.js and synapse-startup.js. All hooks:
    - Use `#!/usr/bin/env node` shebang
    - Read JSON from stdin via process.stdin 'data'/'end' events
    - Wrap ALL logic in top-level try/catch that outputs deny JSON on error (fail-closed for enforcement hooks)
    - Use exit(0) + JSON stdout for structured decisions (NOT exit(2) -- per RESEARCH.md anti-pattern)
    - Import smol-toml from 'smol-toml' (available in root node_modules, verified)
    - Read config files relative to process.cwd() (project root when Claude Code runs hooks)

    **tier-gate.js:**
    - Only triggers on tool_name === 'mcp__synapse__store_decision'
    - Reads packages/framework/config/trust.toml, parses [tier_authority] section
    - Extracts actor from tool_input.actor (default to '' if missing)
    - Extracts requested tier from tool_input.tier (default to 99 if missing -- will always deny)
    - If tier === 0: return permissionDecision "ask" with reason "This is a Tier 0 (Product Strategy) decision. User approval required per trust.toml configuration."
    - If actor not in tier_authority OR requested tier not in actor's tier list: deny with "DENIED: {actor} cannot store Tier {tier} decisions. Allowed tiers: [{list}]. Escalate to the appropriate agent."
    - Otherwise: exit(0) to allow

    **tool-allowlist.js:**
    - Only triggers on tool_name starting with 'mcp__synapse__' (per locked decision: built-in tools not gated)
    - Reads packages/framework/config/agents.toml, parses [agents.{actor}].allowed_tools
    - Extracts actor from tool_input.actor (default to '' if missing)
    - If actor not in agents config OR tool_name not in allowed_tools: deny with "DENIED: {actor} is not authorized to use {tool_name}. Check agents.toml for permitted tools."
    - Otherwise: exit(0) to allow

    **precedent-gate.js:**
    - Only triggers on tool_name === 'mcp__synapse__store_decision'
    - Returns permissionDecision "allow" with additionalContext: "REMINDER: Before storing this decision, verify you have called mcp__synapse__check_precedent with this decision's subject and rationale. If a similar decision already exists, reference it rather than creating a duplicate. This ensures decision consistency across the project."
    - On non-store_decision: exit(0) silently

    **IMPORTANT:** Config paths use `packages/framework/config/` prefix since hooks run from project root (process.cwd()). Use the cwd field from hook input JSON if available, otherwise fall back to process.cwd().
