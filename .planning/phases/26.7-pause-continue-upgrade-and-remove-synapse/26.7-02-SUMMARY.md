## Decisions Made

- `synapse_version` uses upsert semantics: `grep -q "^synapse_version"` to decide append vs sed-replace
- Cross-platform sed: `sed -i.bak "..." && rm -f "*.bak"` works on both GNU sed (Linux) and BSD sed (macOS) — avoids the `-i ''` vs `-i` divergence
- VERSION is already set by install.sh before Section 9b runs — no need to detect it again; local dev mode uses `"local-dev"`, production uses the tag string
- upgrade.md requires explicit user confirmation (yes/no) before running install.sh — no silent upgrades
- Hot-reload detection uses `ls -la` size comparison on `.js` hook files and `settings.json` — any change requires restart; `.md` file changes (agents, commands) are active immediately
