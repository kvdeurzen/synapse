## Decisions Made

- **Fail-open vs fail-closed:** conventional-commit hook uses fail-open because it is an advisory formatting enforcement hook, not a security gate. Output-contract-gate uses fail-closed because bypassing it loses important audit data.
- **[task:id] suffix rejection:** agents sometimes add `[task:abc123]` suffixes for traceability. This is explicitly rejected — traceability belongs in commit body or Synapse task state, not the first line.
- **72-char limit is inclusive:** exactly 72 chars is valid (max first-line length = 72). 73+ is denied.
- **HEREDOC extraction:** HEREDOC-style commits extract the first non-heredoc line. If extraction fails (unusual quoting), the hook passes silently rather than blocking the commit.
