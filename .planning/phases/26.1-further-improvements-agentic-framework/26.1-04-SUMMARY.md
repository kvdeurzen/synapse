## Decisions Made

- **Planner scoped to structure only:** Leaf task descriptions contain WHAT and acceptance criteria. Task Designer adds mock code and file paths. This preserves the Planner's role as a tree-structuring agent while offloading spec work to a dedicated agent.
- **Plan Auditor uses 8 dimensions:** Adapted from GSD Plan-Checker (dimensions 1-8) but applied to Synapse task trees rather than PLAN.md files. Dimension 8 becomes context-budget checking rather than Nyquist compliance.
- **Task Designer model stays opus:** Spec writing requires creative synthesis (understanding codebase patterns, inferring interfaces, writing mock code) — sonnet downgrade would produce lower-quality specs.
- **Task Auditor uses sonnet:** Spec verification is pattern-matching (does the spec have mock code? are file paths explicit?) — sonnet sufficient and more economical for this review-only role.
- **Task Auditor activates Tier 2 AND Tier 3 drafts:** Task Designer may draft both functional choices (Tier 2) and execution-detail choices (Tier 3). Task Auditor activates both, consistent with trust.toml task-auditor = [2, 3].
