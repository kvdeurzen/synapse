Install new dependencies, add the doc_chunks table schema, build the chunking service, and create the activity logging helper. These are the shared infrastructure pieces that all document tools in Plans 02-04 depend on.

Purpose: The chunking pipeline converts raw document text into overlapping, context-prefixed chunks with token counts. The activity logger provides audit trail insertion after every mutation. The doc_chunks table stores embedded document chunks separately from the documents table (which holds full content). All three must exist before any document tool can be implemented.

Output: Extended schema.ts with doc_chunks, new src/services/chunker.ts and src/services/activity-log.ts, plus comprehensive tests for both services.

```typescript
function extractSectionHeader(text: string): string | null {
  // Find the first markdown header in the chunk
  const match = text.match(/^#{1,6}\s+(.+)$/m);
  return match ? match[1].trim() : null;
}
```

**Main chunkDocument function:**
```typescript
export async function chunkDocument(
  title: string,
  content: string,
  category: string,
): Promise<ChunkResult[]> {
  const strategy = getCategoryStrategy(category);

  let rawChunks: string[];

  if (strategy === "semantic_section") {
    const splitter = RecursiveCharacterTextSplitter.fromLanguage("markdown", {
      chunkSize: 2000,     // characters (~500 tokens at ~4 chars/token)
      chunkOverlap: 200,   // ~10% overlap (~50 tokens)
    });
    const docs = await splitter.createDocuments([content]);
    rawChunks = docs.map(d => d.pageContent);
  } else if (strategy === "paragraph") {
    rawChunks = splitByParagraph(content, 2000, 200);
  } else {
    rawChunks = splitFixedSize(content, 2000, 200);
  }

  // If content is too short to chunk, return it as a single chunk
  if (rawChunks.length === 0) {
    rawChunks = [content];
  }

  return rawChunks.map((chunk, index) => {
    const header = extractSectionHeader(chunk) ?? title;
    const contextPrefix = `Document: ${title} | Section: ${header}`;
    const prefixed = `${contextPrefix}\n\n${chunk}`;
    return {
      content: prefixed,
      header,
      tokenCount: countTokens(prefixed),
      chunkIndex: index,
    };
  });
}
```

**Paragraph splitter (hand-rolled per research recommendation):**
```typescript
function splitByParagraph(content: string, maxChars: number, overlap: number): string[] {
  const paragraphs = content.split(/\n\n+/);
  const chunks: string[] = [];
  let current = "";

  for (const para of paragraphs) {
    if (current.length + para.length + 2 > maxChars && current.length > 0) {
      chunks.push(current.trim());
      // Overlap: keep the last `overlap` characters of current chunk
      const overlapText = current.slice(-overlap);
      current = overlapText + "\n\n" + para;
    } else {
      current = current ? current + "\n\n" + para : para;
    }
  }
  if (current.trim().length > 0) {
    chunks.push(current.trim());
  }
  return chunks;
}
```

**Fixed-size splitter (hand-rolled per research recommendation):**
```typescript
function splitFixedSize(content: string, maxChars: number, overlap: number): string[] {
  const chunks: string[] = [];
  let start = 0;
  while (start < content.length) {
    const end = Math.min(start + maxChars, content.length);
    chunks.push(content.slice(start, end));
    if (end >= content.length) break;
    start = end - overlap;
  }
  return chunks;
}
```

**Create `src/services/activity-log.ts`:**

This is a shared helper called by every mutation tool (store_document, update_document, delete_document, link_documents) to insert audit trail rows.

```typescript
import type { Connection } from "@lancedb/lancedb";
import { ulid } from "ulidx";
import { insertBatch } from "../db/batch.js";
import { ActivityLogRowSchema } from "../db/schema.js";
import { logger } from "../logger.js";

/**
 * Log a mutation to the activity_log table.
 *
 * Called AFTER every successful mutation — never before the write succeeds (Research anti-pattern).
 * Uses "agent" as the default actor (MCP SDK does not expose caller identity).
 */
export async function logActivity(
  db: Connection,
  projectId: string,
  action: string,
  targetId: string | null,
  targetType: string | null,
  metadata?: Record<string, unknown>,
): Promise<void> {
  const table = await db.openTable("activity_log");
  await insertBatch(
    table,
    [{
      log_id: ulid(),
      project_id: projectId,
      actor: "agent",
      action,
      target_id: targetId,
      target_type: targetType,
      metadata: metadata ? JSON.stringify(metadata) : null,
      created_at: new Date().toISOString(),
    }],
    ActivityLogRowSchema,
  );
  logger.debug({ projectId, action, targetId, targetType }, "Activity logged");
}
```

Key design decisions:
- Actor is hardcoded to `"agent"` per research recommendation (MCP SDK has no caller identity)
- Called AFTER successful mutation, never before (Research anti-pattern: "Logging before write succeeds")
- Uses the existing `insertBatch` + `ActivityLogRowSchema` pattern
- ULID for log_id (project standard)
- metadata is optional Record serialized to JSON string
