**4. Handle tree-sitter postinstall for workspace context:**

The setup-tree-sitter.js script uses `projectRoot = join(__dirname, "..")` which resolves to `packages/server/`. It then looks for `packages/server/node_modules/tree-sitter/build/Release/...`.

In a Bun workspace, tree-sitter should stay in `packages/server/node_modules/` since only the server depends on it. But if Bun hoists it:

Update `packages/server/scripts/setup-tree-sitter.js` to also check the root node_modules:
```javascript
// After computing builtBinary from projectRoot:
// Also check root node_modules if package-local doesn't exist
const rootBuiltBinary = join(
  projectRoot,
  "../../node_modules/tree-sitter/build/Release/tree_sitter_runtime_binding.node"
);
const rootPrebuildDir = join(
  projectRoot,
  `../../node_modules/tree-sitter/prebuilds/${process.platform}-${process.arch}`
);

// In the Bun block, check both locations:
// if (!existsSync(builtBinary) && !existsSync(rootBuiltBinary)) → skip
// If rootBuiltBinary exists but builtBinary doesn't → use rootBuiltBinary path for prebuildDir too
```

This makes the script resilient to hoisting. If tree-sitter is NOT hoisted (expected case), the existing path works fine.

Default to using Bun instead of Node.js.

## Monorepo Structure

This is a Bun workspace monorepo with two packages:

- `packages/server/` — Synapse MCP server (data layer: LanceDB, embeddings, document/code indexing)

## Testing

- Run all tests: `bun run test`
- Server tests only: `bun run test:server` (or `bun test --cwd packages/server`)
- Framework tests only: `bun run test:framework` (or `bun test --cwd packages/framework`)

## Key Paths

- Agents: `packages/framework/agents/`
- Skills: `packages/framework/skills/`
- Hooks: `packages/framework/hooks/`
- Config: `packages/framework/config/`
