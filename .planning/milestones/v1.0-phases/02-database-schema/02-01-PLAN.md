<objective>
Define all 5 LanceDB table schemas (Arrow + Zod), create the DB connection wrapper, and implement the generic batch insert helper with Zod validation.

Purpose: Establish the schema contracts and data access primitives that init_project and all future phases depend on. The Arrow schemas define the immutable table structure (LanceDB schemas cannot be altered after first write), so they must include all fields including v2 forward-compatibility fields. The Zod schemas serve as the single source of truth for validation on both MCP tool inputs and DB inserts.

Output: Three modules (schema.ts, connection.ts, batch.ts) with unit tests proving schema correctness and validation behavior.
</objective>

**Create `src/db/connection.ts`:**

Simple wrapper around `lancedb.connect()` that auto-creates the database directory.

```typescript
import * as lancedb from "@lancedb/lancedb";
import { mkdirSync } from "node:fs";
import { resolve } from "node:path";
import { logger } from "../logger.js";

export async function connectDb(dbPath: string): Promise<lancedb.Connection> {
  const absPath = resolve(dbPath);
  mkdirSync(absPath, { recursive: true });
  const db = await lancedb.connect(absPath);
  logger.debug({ dbPath: absPath }, "Connected to LanceDB");
  return db;
}
```

Keep it simple — no connection pooling, no caching. Each call returns a fresh connection. Future phases can add caching if needed.

**Create `src/db/batch.ts`:**

Generic batch insert helper that validates rows against a Zod schema before inserting.

```typescript
import type { Table } from "@lancedb/lancedb";
import type { z, ZodTypeAny } from "zod";

export async function insertBatch<S extends ZodTypeAny>(
  table: Table,
  rows: z.infer<S>[],
  zodSchema: S,
): Promise<void> {
  if (rows.length === 0) {
    return; // Nothing to insert
  }
  // Validate every row BEFORE touching the DB — fail fast per user decision
  for (let i = 0; i < rows.length; i++) {
    const result = zodSchema.safeParse(rows[i]);
    if (!result.success) {
      const msgs = result.error.issues.map(
        (e: { path: (string | number)[]; message: string }) => `${e.path.join(".")}: ${e.message}`
      ).join("; ");
      throw new Error(
        `[insertBatch] Row ${i} invalid for table '${table.name}': ${msgs}`
      );
    }
  }
  // Single atomic add — all-or-nothing per user decision
  await table.add(rows as Record<string, unknown>[]);
}
```

Key behaviors per user decisions:
- Validate EVERY row before sending batch (fail fast)
- Fail immediately on error with clear context (table name, row index, error message)
- No retries
- Empty array is a no-op (return early)
