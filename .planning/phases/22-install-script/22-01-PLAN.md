  <name>Task 1: Create install.sh — prerequisite checks, file copy, config generation, merge logic</name>
  <files>install.sh</files>
  <action>
Create install.sh as a bash script with `#!/usr/bin/env bash` and `set -euo pipefail`. The script handles the complete installation flow EXCEPT the smoke test (which delegates to scripts/smoke-test.mjs in Task 2).

**Structure the script in these sections:**

1. **Color and logging setup** — isatty check for colored output. Define log_step (green checkmark), log_warn (yellow warning), log_error (red X), log_info (indented detail). All respect --quiet flag.

2. **Argument parsing** — Support these flags:
   - `--quiet` / `-q`: suppress step-by-step output, only errors and final pass/fail
   - `--smoke-test`: run only the smoke test (skip install), for post-install verification
   - `--local`: install directly into current project only (this is the DEFAULT)
   - `--global`: install to `~/.synapse/` as shared source, then wire into current project
   - `--version TAG`: specify release tag to download (default: latest)
   - `--help`: print usage

3. **Already-installed detection** — Check for `.synapse/` directory. If found and not `--quiet`, prompt "Synapse is already installed. Update to latest? [y/N]". In `--quiet` mode, auto-update. If user declines, exit 0.

4. **Prerequisite checks** (INST-01):
   - Check `bun` on PATH — HARD FAIL if missing (print install URL https://bun.sh). Show version with `bun --version`.
   - Check `ollama` binary on PATH — SOFT WARN if missing (print install URL https://ollama.ai, note "needed for /synapse:map and smoke test").
   - If Ollama binary found, check if running via `curl -sf http://localhost:11434/api/tags`. If running, check for `nomic-embed-text` in the JSON response. If model missing, warn with `ollama pull nomic-embed-text` instruction.
   - Set `OLLAMA_RUNNING=true/false` for later smoke test decision.

5. **Download and extract** — Per user decision, download tagged release tarball from GitHub:
   - URL: `https://github.com/kvdeurzen/synapse/archive/refs/tags/${VERSION}.tar.gz`
   - Extract to temp directory, find the extracted directory (GitHub tarballs extract to `synapse-TAG/`)
   - For development/testing: if script is run from the synapse repo itself (detect via `packages/server/src/index.ts` existing), use local files instead of downloading. This enables testing without a published release.
   - Create `SYNAPSE_SOURCE` variable pointing to the source files (either extracted tarball or local repo)

6. **File copy** (INST-02):
   - Create target directories: `.claude/agents/`, `.claude/hooks/`, `.claude/hooks/lib/`, `.claude/commands/synapse/`, `.claude/skills/`, `.claude/server/`, `.synapse/config/`
   - Copy framework files:
     - `cp` all 11 agent .md files from `$SYNAPSE_SOURCE/packages/framework/agents/` to `.claude/agents/`
     - `cp` all hook .js files from `$SYNAPSE_SOURCE/packages/framework/hooks/` to `.claude/hooks/`
     - `cp -r` the hooks lib/ directory to `.claude/hooks/lib/`
     - `cp` all 5 command .md files from `$SYNAPSE_SOURCE/packages/framework/commands/synapse/` to `.claude/commands/synapse/`
     - `cp -r` all 18 skill directories from `$SYNAPSE_SOURCE/packages/framework/skills/` to `.claude/skills/`
   - Copy server:
     - `cp -r` the entire `$SYNAPSE_SOURCE/packages/server/` to `.claude/server/`
     - Run `cd .claude/server && bun install --production && cd -` to install server dependencies (CRITICAL: native tree-sitter binaries require bun install)
   - Copy config templates (only if not already present — preserve user customizations on re-install):
     - `cp -n` (no-clobber) `$SYNAPSE_SOURCE/packages/framework/config/trust.toml` to `.synapse/config/trust.toml`
     - `cp -n` `$SYNAPSE_SOURCE/packages/framework/config/agents.toml` to `.synapse/config/agents.toml`
     - `cp -n` `$SYNAPSE_SOURCE/packages/framework/config/synapse.toml` to `.synapse/config/synapse.toml`
   - On update (re-install): overwrite agents, hooks, commands, skills, server. Preserve .synapse/config/ files.

7. **Generate/merge settings.json** (INST-02):
   - Define the Synapse hooks JSON structure (see interfaces block above for exact shape).
   - If `.claude/settings.json` does NOT exist: write it fresh.
   - If `.claude/settings.json` ALREADY exists: use `bun -e` inline script to deep-merge. The merge logic:
     - Read existing file
     - For each hook event (SessionStart, PreToolUse, PostToolUse): first REMOVE any existing Synapse hooks (detect by checking command string for `synapse-startup.js`, `tier-gate.js`, `precedent-gate.js`, `tool-allowlist.js`, `audit-log.js`), then append the new Synapse hooks. This prevents duplicate hooks on re-run (Pitfall 7 from RESEARCH).
     - Preserve all non-Synapse hooks
     - Merge statusLine only if not already present or if it's a Synapse statusline
     - Write back with JSON.stringify(merged, null, 2)

8. **Generate/merge .mcp.json** (INST-02):
   - Define the Synapse MCP server entry (see interfaces block above).
   - If `.mcp.json` does NOT exist: write it fresh.
   - If `.mcp.json` ALREADY exists: use `bun -e` to merge — add/replace `mcpServers.synapse` key, preserve all other MCP servers.

9. **Update .gitignore** — Idempotent append of: `.synapse-audit.log`, `.synapse/config/local.toml`, `.synapse/data/`. Check with `grep -qF` before appending each entry. Create .gitignore if it doesn't exist.

10. **Smoke test invocation** — If `OLLAMA_RUNNING=true`, run `bun run scripts/smoke-test.mjs --server-path .claude/server/src/index.ts` (Task 2 creates this file). If it exits 0, log success. If it exits non-zero, log failure but don't fail the install — warn and suggest re-running with `install.sh --smoke-test` after fixing. If Ollama not running, skip smoke test and print instructions to run it later.

11. **Success summary** — Print what was installed (file counts), note that settings.json is gitignored (each developer runs install.sh), and the single next step: "Next: Run /synapse:init in Claude Code".

12. **Cleanup trap** — `trap cleanup EXIT` that removes the temp download directory.

**CRITICAL implementation details:**
- Capture `TARGET_DIR="${TARGET_DIR:-$PWD}"` at script start and use it throughout (Pitfall 5 — curl | bash CWD issue)
- All hook paths MUST use `bun $CLAUDE_PROJECT_DIR/.claude/hooks/` prefix, NEVER `node`, NEVER relative paths
- The `--global` mode: copies source to `~/.synapse/releases/TAG/`, then symlinks or copies from there into the current project. The project still gets its own `.claude/` and `.claude/server/`.
- Make the script executable with `chmod +x install.sh` in Task 2's verify step

**Do NOT implement the smoke-test.mjs file in this task** — that is Task 2. The install.sh script should call it but handle its absence gracefully (skip smoke test if file doesn't exist yet, which won't happen in practice since both files ship together).

<task type="auto">
  <name>Task 2: Create scripts/smoke-test.mjs — MCP stdio smoke test</name>
  <files>scripts/smoke-test.mjs</files>
  <action>
Create scripts/smoke-test.mjs as a standalone Bun script that validates the installed Synapse MCP server works end-to-end. This script is called by install.sh and also by `install.sh --smoke-test` for post-install verification.

**Adapt the pattern from packages/framework/test/helpers/synapse-client.ts** (the exact MCP JSON-RPC protocol implementation).

The script:

1. **Parse arguments** — Accept `--server-path PATH` (default: `.claude/server/src/index.ts`) and `--db PATH` (optional, defaults to creating a temp dir).

2. **Spawn server** — Use `Bun.spawn(["bun", "run", serverPath, "--db", tmpDir])` with stdin/stdout/stderr as pipes. Set env: `OLLAMA_URL=http://localhost:11434`, `EMBED_MODEL=nomic-embed-text`.

3. **Implement JSON-RPC communication** — Port the sendRequest/sendNotification pattern from synapse-client.ts:
   - Background stdout reader that parses newline-delimited JSON-RPC responses
   - Pending promise map keyed by request ID
   - 30-second timeout per request

4. **MCP handshake**:
   - Send `initialize` request (protocolVersion: "2024-11-05", clientInfo: {name: "synapse-install-smoke", version: "1.0"})
   - Send `notifications/initialized` notification
   - Verify initialize response has `result` (not `error`)

5. **Three tool calls** (the smoke test pipeline):
   - `init_project` with `{project_id: "smoke-test-" + Date.now()}` — verify `success: true`
   - `store_document` with `{project_id: <same>, doc_id: "smoke-doc", category: "note", title: "Smoke Test", content: "This is a smoke test document for Synapse installation verification."}` — verify `success: true`
   - `semantic_search` with `{project_id: <same>, query: "smoke test", limit: 1}` — verify `success: true` and results array is non-empty

6. **Result validation** — For each tool call, parse `response.result.content[0].text` as JSON and check `parsed.success === true`. If any returns `success: false`, print the error and exit 1.

7. **Output** — Print each step with status:
   - `  Smoke: MCP handshake... OK`
   - `  Smoke: init_project... OK`
   - `  Smoke: store_document... OK`
   - `  Smoke: semantic_search... OK`
   - `  Smoke: All checks passed` (or `FAILED: <reason>`)

8. **Cleanup** — In a `finally` block: kill the server process, remove the temp DB directory (only if we created it). Use `process.on("exit", cleanup)` as a safety net.

9. **Exit code** — Exit 0 on success, exit 1 on any failure. install.sh reads this exit code.

**CRITICAL implementation details:**
- Use `Bun.spawn` not `child_process.spawn` — this is a Bun script
- The server takes ~1-2 seconds to start — wait for the initialize response before sending tool calls
- The `semantic_search` call requires Ollama to be running (it generates embeddings) — this is by design; the smoke test validates the full pipeline
- Response format: `{jsonrpc: "2.0", id: N, result: {content: [{type: "text", text: "<json>"}]}}`
- Use `parseArgs` from `node:util` for argument parsing (works in Bun)
- The script must be runnable with `bun run scripts/smoke-test.mjs` from the project root
