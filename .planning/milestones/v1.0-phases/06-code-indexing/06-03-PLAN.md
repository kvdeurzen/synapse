  Build the import resolver types and TypeScript-specific path resolution. TDD: tests first.

  **Types:**
  ```typescript
  export interface ImportEdge {
    from: string;       // relative path of importing file (e.g., "src/server.ts")
    to: string;         // relative path of imported file (e.g., "src/utils.ts")
    symbols: string[];  // imported symbol names (may be empty for `import * as X`)
  }

  export interface ResolveOptions {
    fileSet: Set<string>;  // all known relative file paths in the project
    language: "typescript" | "python" | "rust";
    filePath: string;      // relative path of the file being resolved
    imports: string[];     // raw import strings from extractor
  }
  ```

  **Main export — `resolveImports(opts: ResolveOptions): ImportEdge[]`:**
  - Dispatch to language-specific resolver
  - Filter: only include edges where `opts.fileSet.has(edge.to)`
  - Deduplicate edges by (from, to) — merge symbols

  **TypeScript resolver — `resolveTsImport(importPath: string, filePath: string, fileSet: Set<string>): string | null`:**

  For each raw import path (e.g., `"./utils"`, `"../lib/bar"`, `"@lancedb/lancedb"`):
  1. Skip if NOT a relative path (doesn't start with `.` or `..`): return null (external package)
  2. Resolve relative to the importing file's directory: `path.join(path.dirname(filePath), importPath)`
  3. Normalize to forward slashes
  4. Try these extensions in order, returning the first match found in `fileSet`:
     a. As-is (if already has extension)
     b. `${resolved}.ts`
     c. `${resolved}.tsx`
     d. `${resolved}/index.ts`
     e. `${resolved}/index.tsx`
  5. Return the matching path or null

  **Test cases for TypeScript:**
  ```typescript
  const fileSet = new Set([
    "src/utils.ts",
    "src/config.ts",
    "src/lib/bar.tsx",
    "src/components/index.ts",
    "src/types.ts",
    "test/helper.ts",
  ]);

  // Test 1: Relative import with extension resolution
  // From "src/server.ts", import "./utils" → "src/utils.ts"
  resolveImports({
    fileSet, language: "typescript", filePath: "src/server.ts",
    imports: ["./utils"]
  });
  // Expect: [{ from: "src/server.ts", to: "src/utils.ts", symbols: [] }]

  // Test 2: Parent directory import
  // From "src/tools/store.ts", import "../config" → "src/config.ts"

  // Test 3: Index file resolution
  // From "src/app.ts", import "./components" → "src/components/index.ts"

  // Test 4: TSX extension resolution
  // From "src/app.ts", import "./lib/bar" → "src/lib/bar.tsx"

  // Test 5: External package — skipped
  // import "@lancedb/lancedb" → no edge (not relative)

  // Test 6: Non-existent local import — skipped
  // import "./nonexistent" → no edge (not in fileSet)

  // Test 7: Re-export creates edge
  // export { foo } from "./utils" — already in imports list from extractor
  ```

  **What to AVOID:**
  - Do NOT try to resolve `node_modules` imports — skip all non-relative paths
  - Do NOT resolve `.js` extensions — the project uses `.ts`/`.tsx` at source level
  - Do NOT create edges for type-only imports differently — they still represent a dependency

  <files>src/services/code-indexer/import-resolver.ts, test/services/code-indexer/import-resolver.test.ts</files>
  <action>
  Extend import-resolver.ts with Python and Rust resolvers. Append tests.

  **Python resolver — `resolvePyImport(importPath: string, filePath: string, fileSet: Set<string>): string | null`:**

  For each raw import string from the Python extractor:
  1. **Relative imports** (starts with `.`):
     - Count leading dots: 1 = current directory, 2 = parent, 3 = grandparent, etc.
     - Base directory = `path.dirname(filePath)`, then go up `(dots - 1)` levels
     - After dots, the remaining module path uses `.` as separator → replace with `/`
     - Try: `${resolved}.py`, `${resolved}/__init__.py`
  2. **Absolute imports** (no leading dot):
     - These are typically external packages — return null
     - EXCEPTION: if the import path's first segment matches a directory in fileSet that contains `.py` files, resolve it (e.g., `from mypackage.utils import X` where `mypackage/utils.py` exists)
     - Try: `${importPath.replace(/\./g, '/')}.py`, `${importPath.replace(/\./g, '/')}/__init__.py`
     - Only return if found in fileSet; otherwise null (external)

  **Rust resolver — `resolveRustImport(importPath: string, filePath: string, fileSet: Set<string>): string | null`:**

  For each raw import/use string from the Rust extractor:
  1. **`crate::` prefix** → strip `crate::`, replace `::` with `/`, prepend `src/`
     - Try: `src/${path}.rs`, `src/${path}/mod.rs`
  2. **`self::` prefix** → strip `self::`, resolve relative to current file's directory
     - Try: `${dir}/${path}.rs`, `${dir}/${path}/mod.rs`
  3. **`super::` prefix** → strip each `super::`, go up one directory per `super`
     - Try same patterns
  4. **`mod submodule` (from extractor's imports list)** → simple name, no `::` separators
     - Resolve relative to current file: `${dir}/${name}.rs` or `${dir}/${name}/mod.rs`
  5. **Everything else** (external crate like `std::io`, `serde::Deserialize`) → return null

  **Test cases for Python:**

  ```python
  fileSet = new Set([

    "mypackage/utils.py",

    "mypackage/models/__init__.py",

  # Test 4: External import — skipped

  # Test 5: __init__.py resolution
  # From "mypackage/main.py", import ".models" → "mypackage/models/__init__.py"
  ```

  **Test cases for Rust:**
  ```rust
  fileSet = new Set([
    "src/main.rs",
    "src/lib.rs",
    "src/models.rs",
    "src/db/connection.rs",
    "src/db/mod.rs",
    "src/utils.rs",

  # Test 1: crate:: import
  # From "src/main.rs", use "crate::models::User" → "src/models.rs"

  # Test 2: crate:: with nested module
  # From "src/main.rs", use "crate::db::connection" → "src/db/connection.rs"

  # Test 3: mod declaration
  # From "src/lib.rs", mod "utils" → "src/utils.rs"

  # Test 4: self:: import
  # From "src/db/mod.rs", use "self::connection" → "src/db/connection.rs"

  # Test 5: External crate — skipped
  # use "std::io" → no edge
  # use "serde::Deserialize" → no edge

  # Test 6: super:: import
  # From "src/db/connection.rs", use "super::models" → would resolve but src/db/models.rs doesn't exist → no edge
  ```

  **What to AVOID:**
  - Do NOT try to resolve Python `import os` or `import sys` — these are stdlib/external
  - Do NOT resolve Rust paths that start with a known external crate name — only `crate::`, `self::`, `super::`, and simple mod names resolve to local files
  - Do NOT create multiple edges for `use crate::models::{User, Post}` — one edge from file to "src/models.rs" with symbols ["User", "Post"]
  - Do NOT forget to handle `Bun.Glob` path separators on Windows (normalize to forward slashes)
