<objective>
Build project_overview, link_documents, and get_related_documents tools. These complete the Phase 4 tool surface — project_overview provides a dashboard summary (DOC-08), link_documents creates manual relationships between documents (GRAPH-01, GRAPH-02, GRAPH-04), and get_related_documents performs 1-hop graph traversal (GRAPH-03).

Together these complete the document management surface and set up the relationship infrastructure that Phase 5 (search) and Phase 6 (code indexing) will build on.

**Core function `projectOverview()`:**

```typescript
export async function projectOverview(
  dbPath: string,
  projectId: string,
): Promise<ProjectOverviewResult>
```

Flow:
1. Connect to LanceDB, open tables: `documents`, `activity_log`
2. **Total documents:** `await docsTable.countRows(\`project_id = '${projectId}' AND status != 'superseded'\`)`
3. **Counts by category:** For each category in VALID_CATEGORIES:
   ```typescript
   const count = await docsTable.countRows(`project_id = '${projectId}' AND category = '${cat}' AND status != 'superseded'`);
   if (count > 0) countsByCategory[cat] = count;
   ```
   Only include categories with count > 0 to keep the response clean.
4. **Counts by status:** For each status in VALID_STATUSES (except 'superseded'):
   ```typescript
   const count = await docsTable.countRows(`project_id = '${projectId}' AND status = '${status}'`);
   if (count > 0) countsByStatus[status] = count;
   ```
5. **Recent activity (last 5):**
   ```typescript
   const activityRows = await activityTable.query()
     .where(`project_id = '${projectId}'`)
     .limit(5)
     .toArray();
   // Note: LanceDB may not support ORDER BY in .where() — sort in JS after toArray()
   // Sort by created_at descending in JavaScript
   activityRows.sort((a, b) => b.created_at.localeCompare(a.created_at));
   ```
   Map to: `{ action, target_id, target_type, created_at }` (exclude log_id, project_id, actor, metadata for brevity)

   **Important:** LanceDB's .query().where() doesn't have ORDER BY support in the predicate string. Fetch a reasonable number of rows (limit 50) and sort in JS, then take the first 5. This is the pragmatic approach for v1.

   Actually, ULID IDs are monotonically sorted — since log_id is a ULID and activity_log rows are inserted in order, the LAST rows inserted will have the highest ULIDs. However, LanceDB doesn't guarantee insertion order in query results. So: fetch with a generous limit, sort by created_at in JS, take 5.

   ```typescript
   const activityRows = await activityTable.query()
     .where(`project_id = '${projectId}'`)
     .limit(50)
     .toArray();
   const sorted = activityRows
     .sort((a, b) => (b.created_at as string).localeCompare(a.created_at as string))
     .slice(0, 5);
   ```

6. **Key documents (priority >= 4):**
   ```typescript
   const keyDocs = await docsTable.query()
     .where(`project_id = '${projectId}' AND priority >= 4 AND status != 'superseded'`)
     .limit(20)
     .toArray();
   ```
   Map to: `{ doc_id, title, category, status, priority }`

7. Return the assembled result.

});

**Core function `linkDocuments()`:**

```typescript
export async function linkDocuments(
  dbPath: string,
  projectId: string,
  args: LinkDocumentsArgs,
): Promise<LinkDocumentsResult>
```

Flow:
1. Connect to LanceDB, open `relationships` table and `documents` table
2. **Validate both documents exist:**
   - Query documents table for from_id with project_id — if not found, return `{ error: 'DOC_NOT_FOUND', message: 'Source document {from_id} not found' }`
   - Query documents table for to_id with project_id — if not found, return `{ error: 'DOC_NOT_FOUND', message: 'Target document {to_id} not found' }`
3. **Check for duplicate relationships (Research Pitfall 6):**
   ```typescript
   const existing = await relTable.query()
     .where(`from_id = '${fromId}' AND to_id = '${toId}' AND type = '${type}' AND project_id = '${projectId}'`)
     .limit(1)
     .toArray();
   if (existing.length > 0) {
     return { error: 'RELATIONSHIP_EXISTS', message: 'Relationship already exists between {from_id} and {to_id} with type {type}' };
   }
   ```
4. **Create forward relationship:**
   ```typescript
   const now = new Date().toISOString();
   const forwardId = ulid();
   const rows = [{
     relationship_id: forwardId,
     project_id: projectId,
     from_id: args.from_id,
     to_id: args.to_id,
     type: args.type,
     source: "manual",  // GRAPH-04: manual source attribution
     created_at: now,
     metadata: args.metadata ?? null,
   }];
   ```
5. **Create reverse relationship (if bidirectional — GRAPH-02):**
   ```typescript
   const relationshipIds = [forwardId];
   if (args.bidirectional) {
     // Check reverse doesn't already exist
     const reverseExisting = await relTable.query()
       .where(`from_id = '${args.to_id}' AND to_id = '${args.from_id}' AND type = '${type}' AND project_id = '${projectId}'`)
       .limit(1)
       .toArray();
     if (reverseExisting.length === 0) {
       const reverseId = ulid();
       rows.push({
         relationship_id: reverseId,
         project_id: projectId,
         from_id: args.to_id,
         to_id: args.from_id,
         type: args.type,
         source: "manual",
         created_at: now,
         metadata: args.metadata ?? null,
       });
       relationshipIds.push(reverseId);
     }
   }
   ```
6. **Insert all rows:** `await insertBatch(relTable, rows, RelationshipRowSchema)`
7. **Log activity:** `logActivity(db, projectId, 'link_documents', args.from_id, 'relationship', { to_id: args.to_id, type: args.type, bidirectional: args.bidirectional })`
8. Return `{ relationships_created: rows.length, relationship_ids: relationshipIds }`
