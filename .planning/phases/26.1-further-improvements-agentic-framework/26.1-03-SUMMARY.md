## Decisions Made

- Product Researcher is distinct from general Researcher: it analyzes project context (no external web search), stores findings as category "research" documents, and reports to gateway not user. This preserves the gateway-owns-user-communication invariant from Plan 02.
- Architect tier_authority set to [] because architect now uses the draft convention (decision-draft-flow.md) and NEVER calls store_decision directly. The Architecture Auditor owns Tier 1-2 decision activation.
- Architecture Auditor has store_decision capability (tier_authority [1, 2]) making it the sole gatekeeper for architectural decisions entering the project's decision log.
- task-auditor tier_authority [2, 3]: activates Tier 2 functional design decisions from task-designer drafts and (edge case) Tier 3 decisions that require review before activation.
