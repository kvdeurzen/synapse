  <action>
Create two new slash command files following the established frontmatter + process structure from existing commands (init.md, status.md, etc.).

**pause.md:**
- Frontmatter: `name: synapse:pause`, `description: Pause active Synapse work — saves pipeline state for later resumption.`
- allowed-tools: Read, Write, Bash, mcp__synapse__query_documents, mcp__synapse__get_task_tree, mcp__synapse__project_overview
- Objective: Save a full state snapshot of the current work context to `.synapse/state/pause-snapshot.json` so it can be resumed later.
- Process steps:
  1. **Detect pipeline state:** Call `mcp__synapse__project_overview` to check for active task progress. Call `mcp__synapse__get_task_tree` for any active epics. Query RPEV stage documents via `mcp__synapse__query_documents(category: "plan", tags: "rpev-stage")`.
  2. **Detect open refinements:** Call `mcp__synapse__query_documents(category: "refinement")` to find any in-progress refinement sessions.
  3. **If pipeline is active (in-progress tasks exist):** Present the user with two choices:
     - **Graceful drain** — Save snapshot, set `draining: true` conceptually. Message: "Snapshot saved. Current in-flight agents will finish their tasks. Run /synapse:continue in a new session to resume."
     - **Immediate stop** — Save snapshot with `stop_mode: "immediate"`. Capture `git rev-parse HEAD` as `revert_to_commit`. Message: "Snapshot saved. Partial work since commit {sha} may need review. Run /synapse:continue to resume."
  4. **If no pipeline active:** Still save snapshot with `pipeline_active: false`, capturing `session_context` as a brief summary of what the user was working on (from conversation context) and any open refinement doc_ids.
  5. **Capture current git HEAD:** Run `git rev-parse HEAD` via Bash to record `revert_to_commit`.
  6. **Write snapshot:** Use the Write tool to create `.synapse/state/pause-snapshot.json` with the schema: `{ paused_at, pipeline_active, pipeline_stage, active_epic_task_id, active_feature_task_ids, in_progress_task_ids, pool_state_doc_id, pending_refinements, session_context, revert_to_commit, stop_mode }`. Ensure `.synapse/state/` directory exists first via `mkdir -p .synapse/state`.
  7. **Confirm to user:** Display a summary of what was captured and instruct: "Run /synapse:continue to resume this work."
- Attribution section: All Synapse tool calls MUST include `actor: "synapse-gateway"` for audit trail.

**continue.md:**
- Frontmatter: `name: synapse:continue`, `description: Resume paused Synapse work — restores pipeline state from snapshot.`
- allowed-tools: Read, Write, Bash, mcp__synapse__query_documents, mcp__synapse__get_task_tree, mcp__synapse__project_overview, mcp__synapse__get_smart_context
- Objective: Load the pause snapshot, validate it against live state, and resume the RPEV pipeline or session context.
- Process steps:
  1. **Load snapshot:** Read `.synapse/state/pause-snapshot.json`. If file does not exist, inform user: "No paused work found. Use /synapse:pause to save state first."
  2. **Display snapshot summary:** Show what was paused: timestamp, pipeline stage (if active), task IDs, session context.
  3. **Validate against live state:** For each `in_progress_task_ids`, call `mcp__synapse__get_task_tree` to verify the tasks still exist and check their current status. For `pending_refinements`, call `mcp__synapse__query_documents` to verify docs exist. Compare `revert_to_commit` with current `git rev-parse HEAD` to detect if commits have been made since pause.
  4. **Handle stale snapshot:** If tasks have been completed, deleted, or status changed since pause, report what changed and ask user how to proceed: continue with adjusted state or discard snapshot.
  5. **Handle immediate stop mode:** If `stop_mode` was "immediate", check if `revert_to_commit` differs from current HEAD. If so, warn user about partial work commits that may need review. Show the commits since the snapshot: `git log --oneline {revert_to_commit}..HEAD`.
  6. **Resume pipeline:** If `pipeline_active` was true, hand off to the RPEV pipeline by providing the orchestrator with the captured epic/feature/task context. Reconstruct the pipeline entry point based on `pipeline_stage`. The Gateway should dispatch the Orchestrator with the snapshot context included in the handoff.
  7. **Resume non-pipeline session:** If `pipeline_active` was false, display the `session_context` and any pending refinements to help user remember where they left off. Suggest: "You had open refinement sessions. Run /synapse:refine to continue."
  8. **Delete snapshot:** After successful resume (or user discards), delete `.synapse/state/pause-snapshot.json` using Bash `rm`.
- Attribution section: All Synapse tool calls MUST include `actor: "synapse-gateway"` for audit trail.

**Implementation (synapse-startup.js):** Add pause detection AFTER the gateway protocol injection block (after line ~369) and BEFORE the permission mode advisory block. The logic:

```javascript
// --- Pause snapshot detection ---
let pauseContext = "";
try {
  if (projectRoot) {
    const snapshotPath = path.join(projectRoot, ".synapse", "state", "pause-snapshot.json");
    if (fs.existsSync(snapshotPath)) {
      const snapshot = JSON.parse(fs.readFileSync(snapshotPath, "utf8"));
      const stageInfo = snapshot.pipeline_active
        ? `RPEV pipeline at stage ${snapshot.pipeline_stage}`
        : "a session";
      pauseContext = [
        "## Paused Work Detected",
        "",
        `You paused ${stageInfo} on ${new Date(snapshot.paused_at).toLocaleString()}.`,
        "Run /synapse:continue to resume where you left off.",
      ].join("\n");
    }
  }
} catch {
  // Silent fail — never block session start
}
if (pauseContext) contextParts.push(pauseContext);
```

This follows the exact same pattern as the gateway protocol injection — try/catch with silent fail, push to contextParts if non-empty. Position it after gateway protocol and before permission mode advisory so the pause notification appears near the end of context.
