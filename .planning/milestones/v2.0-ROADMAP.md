# Milestone v2.0: Agentic Framework

**Status:** SHIPPED 2026-03-02
**Phases:** 10-14
**Total Plans:** 19

## Overview

Synapse v2.0 delivers the agentic coordination layer on top of the v1.0 data layer: decision precedent tracking with semantic similarity, recursive task hierarchy with cascade status propagation, a Claude Code framework with 10 specialized agents orchestrated via TOML-configured trust tiers, a skill registry with token budgets and tamper detection, hook-based authority enforcement (tier gates, tool allowlists, audit logging), and the Plan-Execute-Validate workflow with wave-based parallel execution. The framework is structured as a Bun workspace monorepo with packages/server/ (data layer) and packages/framework/ (agent layer) after a mid-milestone consolidation from two separate repositories.

## Milestone Summary

### Decimal Phases
- **Phase 13.1** was inserted between Phase 13 and Phase 14 to consolidate synapse-server and synapse-framework from two separate repositories into a single Bun workspace monorepo. This was not originally planned but became necessary as maintaining two repos with cross-references created friction for development and testing. The phase added 2 plans to the milestone total.

### Key Decisions
- Claude Code framework over Agent SDK standalone process (agents are markdown files loaded natively by Claude Code — no custom runtime needed; decouple to Agent SDK later if needed)
- TOML over YAML/JSON for config (human-readable, comment-friendly, typed sections via Zod validation)
- Agent prompts ARE the markdown files (no template engine, no separate prompt store — Claude Code reads agent files directly)
- Trust-Knowledge Matrix as a TOML config (per-domain autonomy levels, not hardcoded — projects can tune their own trust tiers)
- Hook-based enforcement over middleware pattern (PreToolUse/PostToolUse hooks are Claude Code native, no custom interceptor layer)
- Monorepo consolidation mid-milestone (Phase 13.1 inserted — two-repo friction outweighed the cost of restructuring)
- Wave-based parallel execution via Claude Code Task tool (independent tasks in a wave run concurrently, waves are sequential)
- Iteration-3 failure escalation (PEV does not loop forever — 3 decompose/validate attempts, then escalate to user)
- 4 Opus / 6 Sonnet agent split (complex reasoning agents on Opus, execution-focused agents on Sonnet)
- SHA-256 skill hash validation (tampered skill files rejected before injection)

### Issues Resolved
- Two-repo development friction resolved by Phase 13.1 monorepo consolidation
