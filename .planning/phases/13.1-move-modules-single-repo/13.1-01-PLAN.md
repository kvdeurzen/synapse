```json
    "lint": "bun run --filter '*' lint"

```

**Create tsconfig.base.json** (new file at repo root):
Extract common compiler options from both packages. Use the stricter server settings as the base. Options specific to one package stay in the package-level tsconfig.
```json
{
  "compilerOptions": {
    "target": "ESNext",
    "module": "ESNext",
    "moduleResolution": "bundler",
    "strict": true,
    "noUncheckedIndexedAccess": true,
    "exactOptionalPropertyTypes": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "resolveJsonModule": true,
    "isolatedModules": true,
    "types": ["bun-types"]
  }
}
```

Note: `noUncheckedIndexedAccess` and `exactOptionalPropertyTypes` were only in the server's tsconfig. The framework may need these disabled in its own tsconfig if compilation fails. Per RESEARCH.md, test before committing — but start with them in the base and only override in framework if needed.

**Create root biome.json** (new file at repo root, consolidating both configs):
Use the synapse (server) config as base since it is the newer schema version (2.4.4) and has more options. The framework's biome.json (schema 1.9.4, tab indentation) is superseded. Use space indentation (server convention).
```json
{
  "$schema": "https://biomejs.dev/schemas/2.4.4/schema.json",
  "vcs": {
    "enabled": false,
    "clientKind": "git",
    "useIgnoreFile": false
  },
  "formatter": {
    "enabled": true,
    "indentStyle": "space",
    "indentWidth": 2,
    "lineWidth": 100
  },
  "linter": {
    "enabled": true,
    "rules": {
      "recommended": true
    }
  },
  "javascript": {
    "formatter": {
      "quoteStyle": "double"
    }
  },
  "assist": {
    "enabled": true,
    "actions": {
      "source": {
        "organizeImports": "on"
      }
    }
  }
}
```
