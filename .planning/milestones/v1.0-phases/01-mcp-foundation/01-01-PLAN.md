<task type="auto">
  <name>Task 2: Create logger, config loader, and shared types modules</name>
  <files>src/logger.ts, src/config.ts, src/types.ts, test/config.test.ts, test/logger.test.ts</files>

Create the three foundational modules. These are imported by everything that comes after.

**src/types.ts — Shared types:**
```typescript
// Response envelope for all tools — per user decision
export interface ToolResult<T = unknown> {
  success: boolean;
  data?: T;
  error?: string;
}

// Config shape — matches Zod schema in config.ts
export interface SynapseConfig {
  db: string;
  ollamaUrl: string;
  embedModel: string;
  logLevel: 'error' | 'warn' | 'info' | 'debug';
}
```

**src/logger.ts — Pino stderr singleton (CRITICAL for FOUND-07):**
- Import pino, create logger with `pino.destination(2)` — this routes ALL log output to stderr fd 2
- Default level: 'info' (will be overridden after config loads)
- Do NOT import config.ts to avoid circular dependency
- Export: `logger` (singleton), `setLogLevel(level: string): void` (called after config loads)
- Structured JSON format (Pino default)
- Export a `createToolLogger(toolName: string)` function that creates a child logger with `{ correlationId: randomUUID(), tool: toolName }` bindings — this satisfies the "request correlation ID on every tool invocation" locked decision

**src/config.ts — 4-level config loader (CRITICAL for FOUND-02):**
- Import `parseArgs` from 'util' (built-in, works in Bun)
- Import `readFileSync` from 'fs'
- Import `parse as parseToml` from 'smol-toml'
- Import `z` from 'zod'
- Define `ConfigSchema` as a Zod object matching SynapseConfig interface:
  - db: z.string().min(1, 'Database path is required (--db, SYNAPSE_DB_PATH, or synapse.toml db)')
  - ollamaUrl: z.string().url().default('http://localhost:11434')
  - embedModel: z.string().default('nomic-embed-text')
  - logLevel: z.enum(['error', 'warn', 'info', 'debug']).default('info')
- Export `loadConfig()` function that:
  1. Parses CLI args via `parseArgs({ args: Bun.argv.slice(2), options: { db: { type: 'string' }, 'log-level': { type: 'string' }, quiet: { type: 'boolean', short: 'q' } }, strict: false, allowPositionals: false })`
  2. Reads env vars: SYNAPSE_DB_PATH, OLLAMA_URL, EMBED_MODEL
  3. Reads synapse.toml from CWD (optional, catch ENOENT silently, surface parse errors)
  4. Merges with precedence: CLI > env > file > defaults
  5. For --quiet / -q: sets logLevel to 'warn' (lower priority than --log-level)
  6. Calls `ConfigSchema.safeParse(merged)`
  7. On failure: collects ALL errors, writes them to stderr via `console.error` (not console.log!), calls `process.exit(1)`
  8. On success: returns typed SynapseConfig
- IMPORTANT: Use `console.error` for pre-logger output (config errors happen before logger is configured). NEVER use `console.log` anywhere in this file.

**test/logger.test.ts — Logger tests:**

Use `bun:test` (import { describe, test, expect } from 'bun:test') per user decision. Not Vitest.
