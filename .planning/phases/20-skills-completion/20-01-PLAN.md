The skill manifest builder must:

1. Build a combined skill list from two sources:
   - `skills` array from project.toml (already available as `skills` variable at line 81)

   - If SKILL.md not found, log warning to stderr (warn-only, never fail) and use "(no description available)"

3. Build manifest text with this format:
   ```
   ## Available Skills

   Read a skill's full content on demand: `.claude/skills/{skill-name}/SKILL.md`

   - **typescript**: TypeScript conventions for type safety...
   - **testing-strategy** *(role: validator, debugger)*: Language-agnostic testing pyramid...
   ```
   - Project skills listed first (in project.toml order), then role skills alphabetically
   - Role-specific skills tagged with `*(role: agent1, agent2)*`
   - Skills that appear in BOTH project.toml and role_skills are listed once under project skills (no duplicate)

4. Build a `roleSkillsMap` object mapping `skillName -> [agentNames]` from agents.toml role_skills entries.

5. Push skillContext to contextParts (same pattern as other context blocks): `if (skillContext) contextParts.push(skillContext);`

6. Wrap the entire block in try/catch with warn-only error handling:
   ```javascript
   } catch (skillErr) {
     process.stderr.write(`[synapse-startup] Warning: Could not build skill manifest: ${skillErr.message}\n`);
   }
   ```
