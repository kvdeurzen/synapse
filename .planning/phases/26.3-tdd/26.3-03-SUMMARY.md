## Decisions Made

- Orchestrator dispatches test-designer after task-designer (step 8) and before task-auditor (step 10) — the test-designer creates the test artifacts that task-auditor uses for triangulation, so this ordering is required
- BLOCKED escalation routes by type: implementation gap respawns executor, wrong test assumption escalates to planner (who owns the requirement contract), ambiguous spawns task-auditor to adjudicate independently
- Pool Manager step 1b checks executor output document status before proceeding to validator — prevents validator from being dispatched on BLOCKED/NEEDS_CONTEXT tasks, conserving tokens
- Pre-dispatch test verification HALTs wave execution if no test file paths found in task description — enforces that test-designer step was not skipped
