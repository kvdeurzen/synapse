    **## Plan-Execute-Validate (PEV) Workflow**

    Add a section that:
    1. References `@packages/framework/workflows/pev-workflow.md` as the authoritative workflow document
    2. Summarizes the orchestrator's role in PEV: "You are the central coordinator. You spawn all subagents. Subagents cannot spawn other subagents."
    3. Lists the PEV agent roles: Decomposer (planning), Plan Reviewer (verification), Executor (implementation), Validator (checking), Integration Checker (cross-task), Debugger (failure analysis)

    **## Progressive Decomposition Protocol**

    Add instructions for:
    1. Spawn Decomposer subagent via Task tool to decompose epic -> features
    2. Spawn Plan Reviewer subagent to verify decomposition
    3. If reviewer rejects: spawn Decomposer again with reviewer feedback (max 3 cycles, per WFLOW-06)
    4. If rejected after 3 cycles: escalate to user with both decomposer's plan and reviewer's objections
    5. JIT decomposition: decompose features -> tasks only when feature is next to execute

    **## Wave Execution Protocol**

    Add instructions for:
    1. Identify task waves (independent tasks = same wave)
    2. Spawn executor subagents in parallel (Task tool, isolation: "worktree")
    3. Cap at pev.max_parallel_executors concurrent executors
    4. Await all results before moving to validation
    5. Spawn Validator for each completed task
    6. If ANY validation fails: halt wave, enter failure escalation
    7. After all feature tasks pass: spawn Integration Checker
    8. If integration passes: merge feature branch to main
    9. Emit wave checkpoint status block (structured format from pev-workflow.md)

    **## Failure Escalation Protocol**

    Add instructions for (per WFLOW-05):
    1. On executor failure: spawn Debugger subagent with full context handoff:
       - Task spec and acceptance criteria
       - What was attempted (executor's output/error)
       - Error messages and relevant file paths
       - Previous retry context (if any)
    2. Debugger produces diagnostic document via `store_document`
    3. Auto-revert failed task's git changes
    4. Spawn fresh executor with debugger's report in context
    5. Track retry count per task/feature/epic (from pev config)
    6. Escalation ladder: task retries exhausted -> feature escalation -> epic escalation -> user

    **## Rollback Protocol**

    Add instructions for (per WFLOW-08):
    1. Failed tasks: auto-revert via git (worktree discard or git revert)
    2. Passing tasks within a feature: keep (only revert the failed ones)
    3. Task can be reopened: update_task status back to "pending"
    4. Feature rollback: revert all task branches that were merged into the feature branch
    5. Merge to main only after full feature passes integration check

    **## Checkpoint Format**

    Add the wave checkpoint status template:
    ```
    ## Wave {N} Complete -- Feature: {title} ({done}/{total} tasks)

    | Task | Status | Agent | Notes |
    |------|--------|-------|-------|
    | {task_title} | done/failed | executor | {brief} |

    Integration check: PASSED/FAILED
    Next: {next_work} ({count} tasks ready)
    ```

    - Maximum 3 review cycles before escalation to user
    - Focus areas: completeness (all acceptance criteria covered), testability (each task can be validated), dependencies (no circular deps, correct ordering)
