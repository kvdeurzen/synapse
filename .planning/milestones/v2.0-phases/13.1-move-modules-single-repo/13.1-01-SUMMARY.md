## Decisions Made

- Used git subtree add without --squash — preserves all 16 framework commits as accessible merge history (CONTEXT.md requirement)
- Did NOT add `@synapse/server: workspace:*` to framework — framework references server only via MCP stdio protocol, not direct TS imports (clean protocol boundary)
- `noUncheckedIndexedAccess` and `exactOptionalPropertyTypes` placed in base tsconfig — both repos already used these (server had them; framework had equivalent strict:true); framework tsconfig does NOT override them
- `allowImportingTsExtensions: true` and `noEmit: true` remain in framework-specific tsconfig — these are package-specific and incompatible with the server's outDir/declaration setup
- `@types/bun` removed from framework devDependencies — root `bun-types` covers both packages identically
