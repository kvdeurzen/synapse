## What Was Built

### Task 1: Shared Protocol Additions (_synapse-protocol.md)

Added 3 new sections to the shared protocol (43 -> 90 lines):

**Mandatory Context Loading Sequence** — 5-step table every agent must complete before beginning work:
1. Parse SYNAPSE HANDOFF block
2. Load task via `get_task_tree`
3. Load documents from `context_doc_ids` field (with fallback chain)
4. Load decisions from `context_decision_ids` field
5. Begin agent-specific work

**Standard Tag Vocabulary** — 5 tag types for every `store_document` call: agent, type, provides, task, stage.

**Provides Vocabulary** — 11 fixed slugs: architecture, decision-draft, plan, task-spec, implementation, validation-findings, debug-diagnosis, research-findings, audit-findings, integration-report, code-analysis.

### Task 2: Orchestrator Structured Field Migration (synapse-orchestrator.md)

**Upstream Context Routing table** added (9-agent routing table):
- Maps each dispatchable agent to its upstream `context_doc_ids` and `context_decision_ids` sources
- Protocol: read completion report doc_ids, call `update_task` before next agent dispatch

**CONTEXT_REFS references removed** (0 remaining):
- SYNAPSE HANDOFF block `doc_ids` now sources from task's `context_doc_ids` field
- SYNAPSE HANDOFF block `decision_ids` now sources from task's `context_decision_ids` field
- "Building the handoff" updated to read structured fields, not parse text blocks

**Structured field reads added:**
- `output_doc_ids` read from completed executor tasks to pass to validators
- `spec` null-check halt before dispatching executor
