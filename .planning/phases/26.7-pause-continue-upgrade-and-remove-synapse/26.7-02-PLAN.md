**Implementation (install.sh):** Add a new section AFTER section 9 (.gitignore) and BEFORE section 10 (smoke test). Call it "Section 9b: Record synapse_version in project.toml".

```bash
# ============================================================
# Section 9b: Record synapse_version in project.toml
# ============================================================

PROJECT_TOML="$TARGET_DIR/.synapse/config/project.toml"
if [ -f "$PROJECT_TOML" ]; then
  if grep -q "^synapse_version" "$PROJECT_TOML"; then
    # Update existing field — use sed to replace the line
    sed -i "s/^synapse_version = .*/synapse_version = \"$VERSION\"/" "$PROJECT_TOML"
  else
    # Append to file (safe since project.toml has a simple flat structure under [project])
    echo "synapse_version = \"$VERSION\"" >> "$PROJECT_TOML"
  fi
  log_step "Recorded synapse_version = $VERSION in project.toml"
fi
```

The `$VERSION` variable is already set by install.sh (either from `--version` flag, GitHub API, or "local-dev" for dev mode). This works for both fresh installs (if project.toml was just created by /synapse:init) and upgrades (project.toml already exists).

IMPORTANT: On macOS, `sed -i` requires `sed -i ''`. Since the project uses Linux (per env info), plain `sed -i` is correct. However, for cross-platform safety, use: `sed -i.bak "s/..." "$PROJECT_TOML" && rm -f "$PROJECT_TOML.bak"` to handle both GNU and BSD sed.

Create the upgrade slash command following the established frontmatter + process structure.

**upgrade.md:**
- Frontmatter: `name: synapse:upgrade`, `description: Upgrade Synapse to the latest version — shows changelog, applies update, and commits changes.`
- allowed-tools: Read, Write, Bash, Glob
- Objective: Detect current and latest Synapse versions, show the user what changed, apply the upgrade via install.sh, and auto-commit the result.
- Process steps:

  1. **Detect current version:** Read `.synapse/config/project.toml` and extract the `synapse_version` field. If the field does not exist, inform user: "No version recorded — this was likely installed before version tracking was added. The upgrade will proceed and record the new version."

  2. **Detect latest version:** Run via Bash:
     ```bash
     LATEST_TAG=$(curl -sf "https://api.github.com/repos/kvdeurzen/synapse/releases/latest" \
       2>/dev/null | grep '"tag_name"' | head -1 | sed 's/.*"tag_name": *"\([^"]*\)".*/\1/' || echo "")
     if [ -z "$LATEST_TAG" ]; then
       LATEST_TAG=$(curl -sf "https://api.github.com/repos/kvdeurzen/synapse/tags?per_page=1" \
         2>/dev/null | grep '"name"' | head -1 | sed 's/.*"name": *"\([^"]*\)".*/\1/' || echo "")
     fi
     echo "$LATEST_TAG"
     ```
     If no tag found, inform user and abort.

  3. **Compare versions:** If current == latest, inform user: "Already on the latest version ({version})." and stop. If current is empty (no prior version), proceed with install.

  4. **Show changelog diff:** Attempt to fetch the CHANGELOG.md from the target version's tarball or use `git log --oneline {current}..{latest}` if the repo is available. Display a summary of changes between current and target version. If changelog is unavailable, show the GitHub compare URL: `https://github.com/kvdeurzen/synapse/compare/{current}...{latest}`.

  5. **Confirm with user:** Present the version change ({current} -> {latest}) and changelog summary. Ask user to confirm: "Proceed with upgrade? (yes/no)". Do NOT proceed without explicit confirmation.

  6. **Record pre-upgrade state:** Run `git rev-parse HEAD` to capture the commit before upgrade, and snapshot the list of hook files with timestamps: `ls -la .claude/hooks/synapse-*.js` so we can detect which hooks changed.

  7. **Run install.sh:** Execute via Bash:
     ```bash
     bash install.sh --quiet --version {latest_tag}
     ```
     If install.sh is not found at the project root, try to download it from the release tarball. Check exit code — if non-zero, report the error and abort.

  8. **Auto-commit changes:** Stage and commit all changed framework files:
     ```bash
     git add .claude/agents/ .claude/hooks/ .claude/commands/synapse/ .claude/skills/ .claude/server/ .synapse/config/ .mcp.json
     git commit -m "chore(synapse): upgrade to {latest_tag}"
     ```
     If nothing to commit (no changes), inform user: "No files changed — already up to date."

  9. **Hot-reload detection:** Compare the hook files before and after upgrade. If any `.js` files in `.claude/hooks/` changed, or if `.claude/settings.json` changed, display:
     > "Critical files updated (hooks/settings). Please restart Claude Code to apply the new hooks. Agent and command files are already active."

     If only `.md` files changed (agents, commands), display:
     > "Upgrade complete! All changes are active immediately — no restart needed."

  10. **Summary:** Display the upgrade result: old version, new version, files changed count, restart needed (yes/no).

- Attribution section: All Synapse tool calls MUST include `actor: "synapse-gateway"` for audit trail.
