## Accomplishments

- Created `/synapse:pause` command with graceful/immediate stop modes, full snapshot schema (11 fields), MCP pipeline state detection, and confirmation summary
- Created `/synapse:continue` command with snapshot validation (tasks, refinements, git divergence), stale snapshot handling, pipeline resume for all stages, and non-pipeline session resume
- Added pause snapshot detection to synapse-startup.js — injects "## Paused Work Detected" context block with stage info and `/synapse:continue` prompt whenever a snapshot exists
- 4 new TDD test cases covering happy path (pipeline active/inactive), no snapshot, and corrupt snapshot (silent fail)

## Files Created/Modified

- `packages/framework/commands/synapse/pause.md` — 9-step process for detecting pipeline state, presenting graceful/immediate stop choice, capturing snapshot fields, writing JSON to .synapse/state/pause-snapshot.json
- `packages/framework/commands/synapse/continue.md` — 8-step process for loading snapshot, validating live state, handling stale/immediate-stop snapshots, resuming pipeline or session context
- `packages/framework/hooks/synapse-startup.js` — Added 21-line pause detection block (try/catch, existsSync, JSON.parse, contextParts.push) after gateway protocol injection
- `packages/framework/test/unit/hooks.test.ts` — Added `describe("pause snapshot detection")` block with 4 tests and `makeSynapseProject()` helper

## Decisions Made

- File-based snapshot at `.synapse/state/pause-snapshot.json` — consistent with existing `.synapse/state/statusline.json` pattern, no MCP tool dependency for startup detection
- Pause detection positioned after gateway protocol injection, before permission advisory — pause notification is near end of additionalContext so it doesn't interfere with tier/RPEV context ordering
- Silent fail with empty catch block for pause detection — matches the established pattern in gateway protocol injection block
