## Decisions Made

- **`bun install --ignore-scripts`:** tree-sitter@0.25.x `binding.gyp` specifies `-std=c++17` but Node.js 24.13.1 v8config.h explicitly requires C++20 (`#error "C++20 or later required."`). Standard `bun install` fails during native compilation. Workaround: use `--ignore-scripts` to skip the failing tree-sitter build, then `setup-tree-sitter.js` auto-rebuilds via `bunx node-gyp rebuild` with `CXXFLAGS=-std=c++20`.

- **`apache-arrow@18.1.0` added to server dependencies:** `@lancedb/lancedb@0.26.2` declares `apache-arrow` as a `peerDependency` (range `>=15.0.0 <=18.1.0`). Bun workspaces do NOT automatically install peerDependencies. The original standalone server repo likely had it as a transitive dep in the old bun.lock. Now it must be explicit. Added `18.1.0` (exact version already in bun's staging cache, within the allowed range).

- **Lint failures deferred:** Both server (57 errors) and framework (20 errors) fail lint with the consolidated biome 2.4.4 config. The server was already failing lint before the migration. The framework issues are from upgrading from biome 1.9.4 to 2.4.4. These are pre-existing and documented in `deferred-items.md`.
