## Decisions Made

- **code-quality-reviewer is read-only:** No Write/Edit tools, no update_task to change task status. It produces a quality_review document and registers it via output_doc_ids — nothing more. Status transitions remain with the orchestrator.
- **Anti-rationalization entries sourced externally:** Per Phase 26.4-02 locked decision — self-generated rationalizations can be rationalized around; entries must come from Superpowers or OWASP patterns.
- **Review-Reception Protocol uses YAGNI as named check:** Explicit step prevents reviewers from accepting "implement properly" suggestions for unused abstractions.
- **Provides Vocabulary table fully removed:** Not reduced to 13 rows — the entire table was the old source of truth. Keeping both would create drift. The new text contains only a reference to output-contracts.toml plus a current slug count summary.
