## Decisions Made
- Config stripping (steps 5-7) must precede directory deletion (step 8): the `bun -e` inline JSON scripts require the Bun runtime, which might be invoked from `.claude/server/` dependencies if the server were already deleted
- Self-deletion as final step: agent has already loaded this command file into context when executing, so deleting the file does not interrupt the running session
- LanceDB database gets a separate prompt (`delete`/`keep`) rather than being bundled with general file removal — embeddings represent significant indexing work the user may want to preserve
- `.mcp.json` deleted entirely when empty after stripping; `.claude/settings.json` left as `{}` even if empty (Claude Code may expect the file)
- Separate Bash calls for directory deletion steps (not `&&` chaining) so a failure in one step does not prevent subsequent cleanup steps from running
