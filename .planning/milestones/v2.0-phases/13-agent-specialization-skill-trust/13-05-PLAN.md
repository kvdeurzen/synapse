<objective>
Wire agent definitions to skill assignments in agents.toml, sync skills: frontmatter in agent markdown files, and create anti-drift integration tests that verify all config, agents, and skills stay in sync.

Purpose: This is the integration wiring plan. It populates skills in agents.toml for appropriate agents (Executor gets typescript/bun, etc.), verifies that agent markdown `skills:` frontmatter matches agents.toml (the runtime injection path for SKILL-03/SKILL-04), and creates tests that catch drift between agents.toml, trust.toml, agent markdown files, and skill directories. These tests prevent the Pitfall 2 (tools divergence) and Pitfall 5 (skill config drift) documented in the research.

**Key architecture decision:** agents.toml `skills` array is the planning/auditing source of truth. Agent markdown `skills:` frontmatter is the runtime injection path — Claude Code reads it at spawn time and injects full SKILL.md content into agent context. The `src/skills.ts` loader remains useful for programmatic validation (token counting, unreferenced skill warnings) but is NOT the injection mechanism.

Output: Updated agents.toml with skill assignments, verified/synced skills: frontmatter in all agent markdown files, integration test file that validates cross-file consistency, full test suite passing.
</objective>

Agent markdown frontmatter `tools:` field format (comma-separated):
```
tools: Read, Write, Bash, mcp__synapse__store_decision
```

Agent markdown frontmatter `skills:` field format (comma-separated list of skill directory names):
```
skills: [typescript, bun]
```
Claude Code reads this field and injects the full SKILL.md content from each listed skill directory into the agent's context at spawn time. This is the native injection mechanism for SKILL-03. Agents with no skills OMIT the field entirely (not `skills: []`).

agents.toml `allowed_tools` format (TOML array):
```toml
allowed_tools = ["Read", "Write", "Bash", "mcp__synapse__store_decision"]
```

agents.toml `skills` format (TOML array):
```toml
skills = ["typescript", "bun"]
```

The `skills:` frontmatter and agents.toml `skills` array must list the same skill names — just in different serialization formats. agents.toml is the source of truth; frontmatter is the runtime delivery path.
