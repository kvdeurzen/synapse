**Validation schemas:**

```typescript
const VALID_CATEGORIES = [
  "architecture_decision", "design_pattern", "glossary", "code_pattern", "dependency",
  "plan", "task_spec", "requirement",
  "technical_context", "change_record",
  "research", "learning",
] as const;

const VALID_STATUSES = ["draft", "active", "approved", "superseded", "archived"] as const;

const CARRY_FORWARD_CATEGORIES = new Set([
  "architecture_decision", "design_pattern", "glossary", "code_pattern", "dependency",
]);
```

**Input schema (Zod):**
```typescript
const StoreDocumentInputSchema = z.object({
  project_id: z.string().regex(/^[a-z0-9][a-z0-9_-]*$/),
  title: z.string().min(1),
  content: z.string().min(1),
  category: z.enum(VALID_CATEGORIES),
  doc_id: z.string().optional().describe("If provided, creates a new version of the existing document"),
  status: z.enum(VALID_STATUSES).optional().default("draft"),
  phase: z.string().optional(),
  tags: z.string().optional().default("").describe("Pipe-separated tags, e.g. '|typescript|backend|'"),
  priority: z.number().int().min(1).max(5).optional(),
});
```

**Result type:**
```typescript
export interface StoreDocumentResult {
  doc_id: string;
  version: number;
  chunk_count: number;
  token_estimate: number;
}
```

**Core function `storeDocument()`:**

```typescript
export async function storeDocument(
  dbPath: string,
  projectId: string,
  args: StoreDocumentArgs,
  config: SynapseConfig,
): Promise<StoreDocumentResult>
```

Flow:
1. Connect to LanceDB via `connectDb(dbPath)`
2. Open the `documents` table
3. **Version resolution:**
   - If `args.doc_id` is provided, query for existing document: `WHERE doc_id = '{doc_id}' AND project_id = '{projectId}'` ordered by version DESC, limit 1
   - If found: `newVersion = existingVersion + 1`
   - If not found with a user-supplied doc_id: return error `{ error: 'DOC_NOT_FOUND', message: '...' }`
   - If no doc_id provided: generate new ULID, version = 1
4. **Mark old chunks as superseded (if re-versioning):**
   - Open `doc_chunks` table
   - `await docChunksTable.update({ where: \`doc_id = '${docId}' AND project_id = '${projectId}' AND status = 'active'\`, values: { status: 'superseded' } })`
   - Also update the documents table: mark old document rows as superseded: `await documentsTable.update({ where: \`doc_id = '${docId}' AND project_id = '${projectId}' AND status != 'superseded'\`, values: { status: 'superseded', updated_at: now } })`
   - **Carry-forward guard (DOC-10):** When superseding, do NOT set status to 'archived' for carry-forward categories — 'superseded' is acceptable (it means "a newer version exists"), but 'archived' is different (it means "no longer relevant")
5. **Insert new document row:**
   - Use `insertBatch` with `DocumentRowSchema` into documents table
   - Fields: doc_id, project_id, title, content, category, status (from args or 'draft'), version (newVersion), created_at (now), updated_at (now), tags (from args or ''), phase/priority/parent_id/depth/decision_type as provided or null
6. **Chunk and embed:**
   - Call `chunkDocument(title, content, category)` to get ChunkResult[]
   - Extract chunk contents for embedding: `chunks.map(c => c.content)`
   - Call `embed(chunkContents, projectId, config)` — this will throw OllamaUnreachableError if Ollama is down (EMBED-03 write-path fail-fast)
   - **IMPORTANT:** If embed fails, the document row is already inserted but chunks are not. This is acceptable — the document exists in metadata but has no searchable chunks. However, to keep things clean, consider wrapping the entire operation. Per research recommendation: let the embed error propagate — the caller gets a clear error. The document row serves as a record of the attempt.
   - Actually: **roll back the document row insertion if embed fails.** Use a try/catch around the embed call. If it throws, delete the document row we just inserted and re-throw the error. This prevents orphaned document rows.
7. **Insert doc_chunks rows:**
   - For each chunk: create a DocChunkRow with chunk_id (ULID), project_id, doc_id, chunk_index, content (prefixed), vector (from embed), header, version (newVersion), status='active', token_count, created_at
   - Use `insertBatch` with `DocChunkRowSchema`
8. **Calculate token_estimate:** Sum of all chunk tokenCount values
9. **Log activity:** Call `logActivity(db, projectId, 'store_document', docId, 'document', { version: newVersion, chunk_count: chunks.length })`
10. **Return:** `{ doc_id, version: newVersion, chunk_count: chunks.length, token_estimate }`

```typescript
// Seed starter documents for new projects
const starters = starterTypes ?? [...DEFAULT_STARTERS];
let startersSeeded = 0;

// Only seed if at least one table was created (fresh project)
if (tables_created > 0) {
  const docsTable = await db.openTable("documents");
  const now = new Date().toISOString();

  const starterRows = [];
  for (const starterKey of starters) {
    const template = STARTER_TEMPLATES[starterKey];
    if (!template) {
      logger.warn({ starterKey }, "Unknown starter template, skipping");
      continue;
    }
    starterRows.push({
      doc_id: ulid(),
      project_id: projectId,
      title: template.title,
      content: template.content,
      category: template.category,
      status: "active",    // Starters are immediately active
      version: 1,
      created_at: now,
      updated_at: now,
      tags: "",
      phase: null,
      priority: null,
      parent_id: null,
      depth: null,
      decision_type: null,
    });
  }

  if (starterRows.length > 0) {
    await insertBatch(docsTable, starterRows, DocumentRowSchema);
    startersSeeded = starterRows.length;
    logger.info({ projectId, startersSeeded }, "Starter documents seeded");
  }
}
```
