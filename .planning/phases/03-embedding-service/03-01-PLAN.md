**Create `src/errors.ts`:**

Define a typed error class hierarchy so callers can `instanceof`-check each failure mode. Per user decision: "Distinct error types per failure mode — callers can handle each differently."

```typescript
// Base class for all Ollama-related errors
export class OllamaError extends Error {
  constructor(message: string) {
    super(message);
    this.name = 'OllamaError';
  }
}

export class OllamaUnreachableError extends OllamaError {
  constructor(url: string) {
    super(`Ollama unreachable at ${url}. Run: ollama serve`);
    this.name = 'OllamaUnreachableError';
  }
}

export class OllamaModelNotFoundError extends OllamaError {
  constructor(model: string) {
    super(`Model ${model} not found. Run: ollama pull ${model}`);
    this.name = 'OllamaModelNotFoundError';
  }
}

export class OllamaTimeoutError extends OllamaError {
  constructor(url: string, timeoutMs: number) {
    super(`Ollama request timed out after ${timeoutMs}ms at ${url}. Run: ollama serve`);
    this.name = 'OllamaTimeoutError';
  }
}

export class EmbedDimensionError extends OllamaError {
  constructor(model: string, got: number, expected: number) {
    super(
      `Dimension mismatch: model ${model} returned ${got} dimensions, expected ${expected}. ` +
      `This usually means the model changed or a different model is loaded.`
    );
    this.name = 'EmbedDimensionError';
  }
}
```

Every error message includes actionable guidance per user decision. Each class sets `this.name` for reliable error identification.

**Create `src/services/embedder.ts`:**

This is the core embedding service module. It exports the `embed()` function and supporting utilities.

**Constants:**
```typescript
export const EXPECTED_DIMENSIONS = 768;
const BATCH_CHUNK_SIZE = 32;
const MAX_RETRIES = 3;
const BASE_DELAY_MS = 500;
const REQUEST_TIMEOUT_MS = 30_000;
const CACHE_STATS_LOG_INTERVAL = 100; // Log every 100 embed calls
```

**Per-project LRU cache (user decision: 10,000 entries, SHA-256 keys):**

Use a `Map<string, LRUCache<string, number[]>>` for per-project isolation:
- Outer key: `projectId`
- Inner key: `SHA256(modelName + ':' + text)` (via `node:crypto` createHash)
- `max: 10_000` per project cache

Export `getProjectCache(projectId)`, `clearAllCaches()`, and `getCacheStats(projectId)` for testing and monitoring.

Track hits/misses per project cache. Log cache stats to stderr every `CACHE_STATS_LOG_INTERVAL` calls using the project logger (per user decision: "Log cache stats periodically to stderr for debugging and tuning"). Use a simple counter modulo check — do NOT log on every call (Pitfall 5).

**Core HTTP call (`callOllamaEmbed`):**

Internal function that makes a single POST to `/api/embed`:
- Use `fetch()` with `AbortSignal.timeout(30_000)` (per user decision: 30-second timeout)
- Request body: `{ model, input: texts }` — note `input` not `prompt` (Pitfall 1: use `/api/embed`, not `/api/embeddings`)
- Response: `{ embeddings: number[][] }`

**Error classification in the catch block:**
- `TypeError` from fetch → `OllamaUnreachableError` (ECONNREFUSED / DNS failure)
- `DOMException` with `name === 'AbortError'` or `name === 'TimeoutError'` → `OllamaTimeoutError` (Open Question from research: check both names for Bun compatibility)
- After fetch resolves: `!response.ok` with status 404 → `OllamaModelNotFoundError`
- `!response.ok` with other status → generic `OllamaError` with HTTP status and body

Per Pitfall 4: distinguish TypeError (network) from HTTP errors (response.ok check) carefully.

**Retry wrapper (`withRetry`):**

Hand-rolled retry loop (per research: ~20 lines, no external library needed):
- Max 3 attempts
- Exponential backoff: `baseDelayMs * 2^attempt + random jitter (0-100ms)`
- Retry on: `OllamaUnreachableError`, `OllamaTimeoutError`, generic `OllamaError` (HTTP 5xx)
- Do NOT retry: `OllamaModelNotFoundError`, `EmbedDimensionError` — throw immediately (user decision: "Fail fast on definitive errors")

**Dimension assertion:**

After receiving vectors from Ollama, assert each vector has exactly 768 dimensions:
```typescript
function assertDimensions(vectors: number[][], model: string): void {
  for (const vec of vectors) {
    if (vec.length !== EXPECTED_DIMENSIONS) {
      throw new EmbedDimensionError(model, vec.length, EXPECTED_DIMENSIONS);
    }
  }
}
```
This runs BEFORE caching — never cache a bad vector.

**Batch chunking (user decision: ~32 texts):**

Split input arrays into chunks of `BATCH_CHUNK_SIZE` and call `callOllamaEmbed` for each chunk sequentially. Collect results. This prevents Ollama OOM on huge payloads (Pitfall from research: "1000 texts in one call can cause OOM").

**Main `embed()` function:**

```typescript
export async function embed(
  texts: string[],
  projectId: string,
  config: { ollamaUrl: string; embedModel: string },
): Promise<number[][]>
```

Flow:
1. Check cache for each text — partition into cached (have vectors) and uncached (need Ollama call)
2. For uncached texts, chunk into groups of 32
3. For each chunk, call `withRetry(() => callOllamaEmbed(chunk, model, ollamaUrl))`
4. Assert dimensions on all returned vectors
5. Store all new vectors in cache
6. Reassemble results in original input order (critical: maintain index mapping between cached/uncached)
7. Return `number[][]` — one vector per input text, same order as input
