    **plan-auditor.md:**
    - Input: project_id (YES), task_id (YES), context_doc_ids (YES -- plan doc_id + research doc_ids)
    - Output: audit-findings doc (`plan-auditor-audit-{task_id}`, provides: audit-findings) with 8-dimension scores

    - Input: project_id (YES), task_id (YES), context_doc_ids (YES -- plan doc_id), context_decision_ids (YES -- activated decision_ids)
    - Output: writes `spec` field on task via `update_task` (NOT in description), spec-rationale doc (`task-designer-task-spec-{task_id}`, provides: task-spec)

    - Input: project_id (YES), task_id (YES). Reads `spec` field directly from task via get_task_tree. If spec is null: HALT with "Task Designer did not write spec"
    - Output: audit-findings doc (`task-auditor-audit-{task_id}`, provides: audit-findings) with 6-dimension assessment

    **executor.md:**
    - Input: project_id (YES), task_id (YES), context_doc_ids (YES -- set by planner/task-designer), context_decision_ids (NO), spec (YES -- from task.spec field). If spec is null: HALT
    - Output: git commit(s) + executor-summary doc (`executor-implementation-{task_id}`, provides: implementation) + writes `output_doc_ids` on task via `update_task`
    - CRITICAL: after storing executor-summary, call `update_task(output_doc_ids: '["executor-implementation-{task_id}"]')` to register produced output
